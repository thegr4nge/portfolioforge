# Milestone v1.0: MVP

**Status:** SHIPPED 2026-02-20
**Phases:** 1-8
**Total Plans:** 27

## Overview

PortfolioForge delivers a complete CLI portfolio intelligence tool across 8 phases, following the natural dependency chain of financial analysis: reliable market data feeds into backtesting and risk measurement, which feeds into portfolio optimisation, which calibrates Monte Carlo projections, which combines with contribution modelling and stress testing, and finally gets polished CLI output and export capabilities. Each phase delivers a standalone, verifiable analytical capability that builds on the previous.

## Milestone Summary

**Decimal Phases:** None (clean linear execution)

**Key Decisions:**
- Pydantic BaseModel for all domain models (validation, serialization)
- Service layer pattern: services/ orchestrates engines/ + data/ + models/, output/ handles rich rendering
- Engine functions take pandas primitives (DataFrame, ndarray) not Pydantic models -- keeps engines pure
- yfinance with sequential fetching + rate limiting (not batch mode)
- Frankfurter API for FX rates (free, no key, ECB data)
- Historical VaR method (np.percentile) -- no scipy needed
- Ledoit-Wolf shrinkage for covariance estimation (PyPortfolioOpt)
- Log returns for Monte Carlo parameter estimation
- Lazy imports in CLI commands for faster startup
- Duplicated test helpers per test file (no cross-test imports)

**Issues Resolved:**
- yfinance repair=True incompatible with numpy 2.x/pandas 3.0 -- disabled without impact
- Rich markup conflicts with bracket text in explanations -- resolved with highlight=False
- Phase 2 placeholder messages for future commands -- cleaned up when those commands were implemented

**Issues Deferred:**
- 30-year data availability varies by ticker (no fallback strategy implemented)
- URTH (MSCI World proxy) only goes back to 2012 (limited for long backtests)
- Mixed import styles across phases (top-level vs lazy) -- cosmetic inconsistency
- _fetch_all used cross-module with private underscore convention

**Technical Debt Incurred:**
- No dedicated stress service tests (engine tests cover the logic)
- 1 cosmetic lint issue (quoted type annotation in models/optimise.py)
