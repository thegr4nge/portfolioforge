    from dataclasses import dataclass
    from datetime import date
    import sqlite3
    from loguru import logger

    **`@dataclass(frozen=True) class DateRange`**:
    - `from_date: date`
    - `to_date: date`
    - `def days(self) -> int: return (self.to_date - self.from_date).days + 1`

    **`class CoverageTracker`**:

    `__init__(self, conn: sqlite3.Connection) -> None`: stores conn.

    **`def get_covered_ranges(self, security_id: int, data_type: str, source: str) -> list[DateRange]`**:
    - Query: `SELECT from_date, to_date FROM ingestion_coverage WHERE security_id=? AND data_type=? AND source=? ORDER BY from_date`
    - Return sorted list of DateRange (from_date/to_date as date objects parsed from ISO strings)

    **`def get_gaps(self, security_id: int, data_type: str, source: str, requested_from: date, requested_to: date) -> list[DateRange]`**:
    - Core gap detection algorithm:
      1. Get covered ranges for this security+data_type+source
      2. If no covered ranges: return `[DateRange(requested_from, requested_to)]` (whole range is a gap)
      3. Walk covered ranges as intervals and find uncovered sub-intervals within [requested_from, requested_to]
      4. Algorithm:
         - current_start = requested_from
         - For each covered DateRange (sorted by from_date):
           - If covered.from_date > current_start: gap found → DateRange(current_start, covered.from_date - 1 day)
           - Advance current_start to max(current_start, covered.to_date + 1 day)
         - If current_start <= requested_to: gap at end → DateRange(current_start, requested_to)
    - Return list of gap DateRanges (may be empty if fully covered)
    - Log: `logger.debug("security_id={} data_type={}: {} gap(s) found in [{}, {}]", ...)`

    **`def record_coverage(self, security_id: int, data_type: str, source: str, from_date: date, to_date: date, records: int) -> None`**:
    - INSERT OR REPLACE into ingestion_coverage
    - Use `ON CONFLICT(...) DO UPDATE SET records=excluded.records, fetched_at=datetime('now')`
    - This is called after a successful fetch to record what was retrieved

    Type annotations everywhere. No pandas. No ORM.

    **`class AdjustmentCalculator`**:

    `__init__(self, conn: sqlite3.Connection) -> None`: stores conn.

    **`def recalculate_for_split(self, security_id: int, split: SplitRecord) -> int`**:
    - The adjustment factor for rows BEFORE the split ex_date is: `split.split_from / split.split_to`
      - AAPL 4:1 forward split (split_from=1, split_to=4): factor = 1/4 = 0.25
      - Historical $400 price → adj_close = $400 * 0.25 = $100
    - SQL (single UPDATE for all historical rows):
      ```sql
      UPDATE ohlcv
      SET
          adj_factor = adj_factor * ?,          -- multiply existing factor by new split factor
          adj_close  = close * (adj_factor * ?) -- recompute from raw close * cumulative factor
      WHERE security_id = ? AND date < ?
      ```
    - Parameters: `(factor, factor, security_id, split.ex_date)`
    - Returns `cursor.rowcount` (number of rows updated)
    - Log: `logger.info("Split {}/{} on {}: updated {} historical rows for security_id={}", ...)`
    - Commits with `with self.conn:` context manager

    **`def recalculate_all_splits(self, security_id: int) -> int`**:
    - Fetch all splits for this security ordered by ex_date ASC
    - Reset all adj_factor to 1.0 and adj_close = close first (full recalculation):
      ```sql
      UPDATE ohlcv SET adj_factor = 1.0, adj_close = close WHERE security_id = ?
      ```
    - Apply each split in chronological order using recalculate_for_split
    - Returns total rows updated
    - Use this when splits are backfilled or corrected — do NOT use on every ingestion (use recalculate_for_split for incremental)

    **`def get_existing_splits(self, security_id: int) -> list[SplitRecord]`**:
    - Query splits table: `SELECT security_id, ex_date, split_from, split_to FROM splits WHERE security_id=? ORDER BY ex_date`
    - Return list[SplitRecord]

    Type annotations everywhere. Log at info level for split operations (they are significant events).
  </action>

from src.market_data.pipeline.adjuster import AdjustmentCalculator

# Insert AAPL security

    OHLCVRecord(security_id=sec_id, date='2020-08-28', open=498.0, high=502.0, low=497.0, close=499.0, volume=10000000, adj_close=499.0),
