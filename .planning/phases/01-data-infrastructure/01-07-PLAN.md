    **`@dataclass class ValidationReport`**:
    - `security_id: int`
    - `total_rows: int`
    - `flagged_rows: int`
    - `flags_by_type: dict[str, int]` — {"ZERO_VOLUME": N, "OHLC_VIOLATION": M, ...}
    - `def is_clean(self) -> bool: return self.flagged_rows == 0`

    **`class ValidationSuite`**:

    `__init__(self, conn: sqlite3.Connection) -> None`:
    - `self._conn = conn`
    - `self._writer = DatabaseWriter(conn)`

    **`def validate(self, security_id: int) -> ValidationReport`**:
    - Fetch all OHLCV rows for security_id: `SELECT date, open, high, low, close, volume, adj_factor FROM ohlcv WHERE security_id=? ORDER BY date`
    - Fetch all split ex_dates for this security: used to exclude split dates from PRICE_SPIKE check
    - For each row, accumulate flags by running all 6 checks
    - Call `self._writer.update_quality_flags(security_id, date, int(combined_flags))` for rows with any flags
    - For rows with no flags AND existing quality_flags != 0: reset to 0 (re-validate clears stale flags)
    - Build and return ValidationReport

    **Check implementations (each as a private method):**

    `def _check_zero_volume(self, volume: int) -> bool`:
    - Returns True (flag should be set) if volume == 0

    `def _check_ohlc_violation(self, open_: float, high: float, low: float, close: float) -> bool`:
    - Returns True if ANY of: low > open_, low > close, high < open_, high < close
    - This is: `low > min(open_, close) or high < max(open_, close)` — simplified form

    `def _check_price_spike(self, prev_close: float | None, close: float, date: str, split_dates: set[str]) -> bool`:
    - Returns False if prev_close is None (first row, no comparison) or date is a known split ex_date
    - Change = abs(close - prev_close) / prev_close
    - Returns True if change > 0.50 (50%)

    `def _check_gap_adjacent(self, dates: list[str], idx: int) -> bool`:
    - For the row at `idx`, check if there is a gap > 5 calendar days between this date and prev_date OR this date and next_date
    - Calendar day threshold: 5 days accounts for weekends (Fri→Mon = 3 days, so gaps > 5 are abnormal)
    - Returns True if either adjacent gap > 5 calendar days
    - Note: Uses calendar days, not trading days (we don't have a trading calendar in Phase 1)

    `def _check_fx_estimated(self, date: str, currency: str) -> bool`:
    - If currency == "USD": return False (no FX needed for USD)
    - Query fx_rates table for exact date match
    - Returns True if no exact match exists (FX rate would need to be interpolated)

    `def _check_adjusted_estimate(self, adj_factor: float, split_dates: set[str], date: str) -> bool`:
    - Returns True if adj_factor != 1.0 AND no split ex_date exists at or before this date in the splits table
    - This catches cases where adj_factor was applied without a corresponding split record

    The validate() method ORs all True check results into a single `QualityFlag` value per row. Only call `update_quality_flags` when `int(combined_flags) != current_quality_flags` (avoid unnecessary DB writes).

    Log summary: `logger.info("security_id={}: validated {} rows, {} flagged", ...)`

from src.market_data.db.schema import run_migrations
from src.market_data.db.writer import DatabaseWriter

from src.market_data.quality.validator import ValidationSuite

w = DatabaseWriter(conn)
sec_id = w.upsert_security(SecurityRecord(ticker='TEST', exchange='NYSE', currency='USD'))
