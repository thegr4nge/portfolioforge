    **data/cache.py** — SQLite-based price cache with TTL eviction:

    Create a `PriceCache` class that manages a SQLite database for caching market data.

    **Schema** (created on first use via `_init_db`):
    ```sql
    CREATE TABLE IF NOT EXISTS price_cache (
        ticker TEXT NOT NULL,
        date TEXT NOT NULL,        -- ISO format YYYY-MM-DD
        open REAL,
        high REAL,
        low REAL,
        close REAL NOT NULL,
        volume INTEGER,
        currency TEXT NOT NULL,    -- original currency
        fetched_at TEXT NOT NULL,  -- ISO timestamp of when this row was cached
        PRIMARY KEY (ticker, date)
    );

    CREATE TABLE IF NOT EXISTS fx_cache (
        base TEXT NOT NULL,
        quote TEXT NOT NULL,
        date TEXT NOT NULL,
        rate REAL NOT NULL,
        fetched_at TEXT NOT NULL,
        PRIMARY KEY (base, quote, date)
    );

    CREATE INDEX IF NOT EXISTS idx_price_ticker ON price_cache(ticker);
    CREATE INDEX IF NOT EXISTS idx_price_fetched ON price_cache(fetched_at);
    CREATE INDEX IF NOT EXISTS idx_fx_fetched ON fx_cache(fetched_at);
    ```

    **Methods:**
    - `__init__(self, db_path: Path | None = None)` — defaults to `config.CACHE_DB_PATH`, creates parent dirs, calls `_init_db()`
    - `get_prices(self, ticker: str, start: date, end: date) -> pd.DataFrame | None` — returns cached prices as DataFrame (columns: date, open, high, low, close, volume, currency) if ALL dates in range are cached AND fetched_at is within TTL. Returns None if cache miss or stale.
    - `store_prices(self, ticker: str, df: pd.DataFrame, currency: str) -> None` — upserts price rows. The DataFrame should have a DatetimeIndex and columns for open, high, low, close, volume. Use `INSERT OR REPLACE`.
    - `get_fx_rates(self, base: str, quote: str, start: date, end: date) -> pd.DataFrame | None` — same pattern for FX cache, uses FX_CACHE_TTL_HOURS.
    - `store_fx_rates(self, base: str, quote: str, df: pd.DataFrame) -> None` — store FX rates (DatetimeIndex, rate column).
    - `evict_stale(self) -> int` — delete all rows where fetched_at is older than TTL. Return count of deleted rows.
    - `clear(self) -> None` — drop all data (useful for testing).

    Use `sqlite3` from stdlib. Use context manager for connections. Use `datetime.now(UTC)` for fetched_at timestamps. Use WAL mode for better concurrent read performance: `PRAGMA journal_mode=WAL`.

    **Important:** Cache checks should verify that the requested date range is MOSTLY covered (allow up to 10% gaps for weekends/holidays). Don't require every calendar day to be present -- market data only has trading days.

    **data/validators.py** — Ticker validation and data integrity:
    - `validate_ticker_format(ticker: str) -> bool` — basic format check: alphanumeric with optional market suffix (.AX, .L, .PA, .DE), or index prefix (^). Reject empty, whitespace, special chars.
    - `validate_price_data(df: pd.DataFrame, ticker: str) -> list[str]` — integrity checks on downloaded data. Return list of warning strings (empty = clean). Checks:
      - No more than 5% NaN values in close column
      - No single-day price moves >50% without a split (log as warning, don't reject)
      - At least 20 trading days of data (reject if fewer)
      - Close prices are all positive
    - `normalize_ticker(ticker: str) -> str` — uppercase, strip whitespace. Convert common aliases: "SP500" -> "^GSPC", "ASX200" -> "^AXJO".

    **data/fetcher.py** — yfinance wrapper with caching:

    Create a module with these functions:

    - `fetch_ticker_data(ticker: str, period_years: int = 10, cache: PriceCache | None = None) -> FetchResult`:
      1. Normalize and validate ticker format. If invalid, return FetchResult with error.
      2. Compute start/end dates: end = today, start = end - period_years.
      3. Check cache. If cache hit, return FetchResult with from_cache=True.
      4. Call yfinance: `yf.download(ticker, start=start, end=end, auto_adjust=True, repair=True, progress=False)`. Use `auto_adjust=True` to get adjusted prices directly in the Close column (yfinance 1.x removed Adj Close, auto_adjust handles it).
      5. Validate downloaded data via `validate_price_data()`. If critical issues, return error.
      6. Store in cache.
      7. Convert DataFrame to PriceData model and return in FetchResult.
      - Wrap all yfinance calls in try/except. Catch `Exception` broadly (yfinance raises various undocumented exceptions). Map to clear error messages:
        - Empty DataFrame -> "No data found for {ticker}. The ticker may be invalid or delisted."
        - ConnectionError -> "Network error fetching {ticker}. Check your internet connection."
        - Any other -> "Unexpected error fetching {ticker}: {str(e)}"

    - `fetch_multiple(tickers: list[str], period_years: int = 10, cache: PriceCache | None = None) -> list[FetchResult]`:
      - Fetch each ticker individually (not batch — yfinance batch download has unreliable error handling per-ticker).
      - Add a `time.sleep(0.3)` between requests for rate limiting.
      - Return list of FetchResult (some may have errors, that is OK).
      - Log progress to stderr using `rich.console.Console(stderr=True)`.
