<feature>
  <name>Performance metrics module</name>
  <files>src/market_data/backtest/metrics.py, tests/test_backtest_metrics.py</files>
  <behavior>
All functions accept a `pd.Series` with a `DatetimeIndex` or `date` index and float values representing portfolio value at each date.

**total_return(equity_curve) -> float**
- Cases:
  - [10000, 11000] → 0.10 (10% gain)
  - [10000, 9000] → -0.10 (-10% loss)
  - [10000, 10000] → 0.0 (flat)
  - Formula: (final / initial) - 1.0

**cagr(equity_curve) -> float**
- Cases:
  - [10000] over 365 days ending at 11000 → ~0.10 (10% annual)
  - Uses calendar days / 365.25 for years; years <= 0 returns 0.0
  - A single-day curve (0 elapsed days) returns 0.0 safely
  - Formula: (final / initial) ^ (1 / years) - 1

**max_drawdown(equity_curve) -> float**
- Cases:
  - [100, 110, 90, 95] → -0.1818... (90/110 - 1, peak=110 at day 2)
  - [100, 110, 120] → 0.0 (no drawdown, always increasing — actually returns 0.0)
  - [100, 50, 80] → -0.50 (50 / 100 - 1 = -0.50)
  - Returns a negative float (or 0.0 if no drawdown occurred)
  - Formula: min((equity - cummax) / cummax)

**sharpe_ratio(equity_curve, risk_free_rate=0.0) -> float**
- Cases:
  - Flat curve [10000, 10000, 10000] → 0.0 (std dev = 0)
  - Strong upward trend → positive value
  - risk_free_rate=0.0 is default — no RBA data dependency
  - Formula: (mean_daily_excess_return / std_daily_return) * sqrt(252)
  - daily_rf = (1 + risk_free_rate) ^ (1/252) - 1
  - Uses pct_change().dropna() for daily returns

**Fixture construction:** build equity curves as pd.Series with pd.date_range index. Numeric assertions use `pytest.approx(rel=1e-4)` for floating point.
