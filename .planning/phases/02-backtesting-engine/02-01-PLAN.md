<task type="auto">
  <name>Task 2: Backtest engine computation + tests</name>
  <files>src/portfolioforge/engines/backtest.py, tests/portfolioforge/test_backtest_engine.py</files>
  <action>
Create `src/portfolioforge/engines/backtest.py` with these pure functions (NO I/O, NO imports of rich/plotext/fetcher):

1. **`align_price_data(price_data_list: list[PriceData]) -> pd.DataFrame`**
   - Convert list of PriceData models to a single DataFrame. Use `aud_close` if available, else `close_prices`.
   - Build each as a pd.Series with `pd.to_datetime(dates)` index, named by ticker.
   - Concatenate with `pd.concat(series_list, axis=1, join="inner")` for overlapping dates only.
   - Sort index, dropna. Return DataFrame (columns=tickers, index=DatetimeIndex).

2. **`compute_cumulative_returns(prices: pd.DataFrame, weights: np.ndarray, rebalance_freq: str | None) -> pd.Series`**
   - If rebalance_freq is None (buy-and-hold): normalise prices (divide by first row), weight, sum across columns. Vectorised.
   - If rebalance_freq is set (e.g. "MS", "QS", "YS"): iterate daily returns, reset weights at rebalance dates.
     - Get rebalance dates via `daily_returns.resample(freq).first().index` converted to a set.
     - Track portfolio_value starting at 1.0, drift weights between rebalance dates.
     - Return pd.Series with DatetimeIndex.
   - Follow the exact code patterns from RESEARCH.md (they were verified working).

3. **`compute_metrics(cumulative: pd.Series, risk_free_rate: float = 0.04) -> dict[str, float]`**
   - total_return: `cumulative.iloc[-1] / cumulative.iloc[0] - 1`
   - annualised_return: `(1 + total_return) ** (252 / n_days) - 1`
   - volatility: `daily_returns.std() * np.sqrt(252)`
   - sharpe_ratio: `(ann_return - risk_free_rate) / volatility` (0.0 if volatility is 0)
   - max_drawdown: `(cumulative / cumulative.cummax() - 1).min()`
   - Return dict with these 5 keys.

4. **`compute_final_weights(prices: pd.DataFrame, weights: np.ndarray) -> list[float]`**
   - Compute ending effective weights by normalising each ticker's total growth.
   - `growth = prices.iloc[-1] / prices.iloc[0]`, `drifted = weights * growth.values`, `drifted / drifted.sum()`.

Then create `tests/portfolioforge/test_backtest_engine.py`:

- **test_rebalanced_returns**: Verify rebalancing produces different results than buy-and-hold for a diverging 2-ticker portfolio.
- **test_compute_metrics**: Create a simple cumulative series (e.g., [1.0, 1.01, 1.02, 0.99, 1.03]) and verify total_return, max_drawdown are correct.
- **test_compute_final_weights**: Verify that a 50/50 portfolio where one ticker doubles and the other stays flat results in ~[0.667, 0.333] final weights.

Use `from portfolioforge.models.portfolio import PriceData` and `from portfolioforge.models.types import Currency` in tests. Use pytest fixtures where appropriate.
