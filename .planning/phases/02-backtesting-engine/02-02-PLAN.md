**Create `src/portfolioforge/services/backtest.py`:**

Function `run_backtest(config: BacktestConfig) -> BacktestResult`:

1. Determine date range: if start_date/end_date provided, use those. Otherwise compute from period_years (end=today, start=today - period_years*365 days).
2. Fetch price data for all config.tickers using `fetch_with_fx()` from `data.fetcher`. Use a PriceCache instance. Use an fx_cache dict for batch FX (same pattern as fetch_multiple).
3. Fetch benchmark data for config.benchmarks (default to `config.DEFAULT_BENCHMARKS` keys' tickers if config.benchmarks is empty).
4. Collect successful FetchResults. If any ticker fails, raise ValueError with the error message.
5. Call `align_price_data()` on portfolio tickers' PriceData. Get the aligned DataFrame.
6. If benchmarks fetched successfully, align each benchmark separately against the portfolio date range (inner join with portfolio dates).
7. Warn (via rich console to stderr) if effective date range is shorter than requested (e.g., "Note: Effective period is 2012-01-03 to 2025-02-05 (URTH data only available from 2012)").
8. Build weights as np.ndarray from config.weights.
9. Call `compute_cumulative_returns()` for the portfolio.
10. Call `compute_cumulative_returns()` for each benchmark (weight=1.0, single asset, no rebalancing).
11. Call `compute_metrics()` on portfolio cumulative and each benchmark cumulative.
12. Call `compute_final_weights()` for the portfolio.
13. Construct and return BacktestResult with all data.

**Create `src/portfolioforge/output/backtest.py`:**

Function `render_backtest_results(result: BacktestResult, console: Console) -> None`:

1. Print a rich Panel header: "Backtest Results: {portfolio_name}" with subtitle showing date range and rebalance frequency.
2. Print a **Performance Summary** rich Table comparing portfolio vs each benchmark:
   - Columns: Metric | Portfolio | {benchmark names...}
   - Rows: Total Return, Annualised Return, Max Drawdown, Volatility, Sharpe Ratio
   - Format percentages with `_color_pct()` helper: green for positive, red for negative. Format as "+12.34%" or "-5.67%".
   - Sharpe ratio as plain float with 2 decimal places.
3. Print a **Portfolio Allocation** table showing initial vs final (drifted) weights:
   - Columns: Ticker | Initial Weight | Final Weight | Drift
   - Color drift: green if small (<5pp), yellow if moderate (5-15pp), red if large (>15pp).
4. Print a summary line: "Period: X years, Y trading days | Rebalancing: {freq}"

Use `from rich.console import Console`, `from rich.table import Table`, `from rich.panel import Panel`. Follow the rich patterns already in `cli.py`.

```python
@app.command()
def backtest(
    ticker: Annotated[
        list[str],
        typer.Option(help="Ticker:weight pairs (e.g. AAPL:0.4 MSFT:0.6)"),
    ],
    period: Annotated[
        str,
        typer.Option(help="Lookback period (e.g. 10y, 5y)"),
    ] = f"{config.DEFAULT_PERIOD_YEARS}y",
    rebalance: Annotated[
        str,
        typer.Option(help="Rebalancing frequency: monthly, quarterly, annually, never"),
    ] = "never",
    benchmarks: Annotated[
        bool,
        typer.Option("--benchmarks/--no-benchmarks", help="Compare against benchmarks"),
    ] = True,
) -> None:
```
