    Create `src/market_data/backtest/tax/engine.py` with the `run_backtest_tax()` function.

    **Function signature (locked in CONTEXT.md):**
    ```python
    def run_backtest_tax(
        portfolio: dict[str, float],
        start: date,
        end: date,
        rebalance: str,
        initial_capital: float = 10_000.0,
        benchmark: str = "STW.AX",
        db_path: str = "data/market.db",
        risk_free_rate: float = 0.0,
        marginal_tax_rate: float = 0.325,
        franking_credits: dict[str, float] | None = None,
    ) -> TaxAwareResult:
    ```

    **Implementation order inside the function:**
    1. Call `run_backtest(portfolio, start, end, rebalance, initial_capital, benchmark, db_path, risk_free_rate)` → get `bt_result: BacktestResult`
    2. Open a DB connection: `conn = get_connection(db_path)`
    3. Determine portfolio currency from the securities table (single query: `SELECT DISTINCT currency FROM securities WHERE ticker IN (...)`) — needed to know if FX conversion applies
    4. Replay trades through CostBasisLedger:
       - For each trade in `bt_result.trades` (sorted by date, ascending — they should already be):
         - If action == "BUY": construct `OpenLot`; if currency is "AUD", cost_basis_usd=None and cost_basis_aud = shares × price + trade.cost (include brokerage per RESEARCH.md open question 4); if currency is "USD": fx_rate = get_aud_usd_rate(conn, trade.date); cost_basis_usd = shares × price + trade.cost; cost_basis_aud = usd_to_aud(cost_basis_usd, fx_rate). Call ledger.buy(ticker, lot).
         - If action == "SELL": similar FX logic for proceeds. proceeds_usd = shares × price - trade.cost (net of brokerage) if USD; proceeds_aud = usd_to_aud or direct. Call ledger.sell(ticker, shares, trade.date) → list[DisposedLot].
         - For each DisposedLot: compute gain_aud = proceeds_aud - cost_basis_aud; set discount_applied = qualifies_for_discount(acquired_date, disposed_date); replace the DisposedLot with a new one including gain_aud and discount_applied (DisposedLot is frozen — construct a new one with all fields).
    5. Load dividend records from DB: `load_dividends(conn, list(portfolio.keys()), start, end)` (implement this function in tax/engine.py using the SQL pattern from RESEARCH.md)
    6. For each ticker, check if franking credits apply per dividend event:
       - For each DividendRecord: resolve franking_pct = resolve_franking_pct(record.ticker, franking_credits); determine total_credits for the year to check $5000 threshold; if should_apply_45_day_rule(total_credits): check satisfies_45_day_rule() per lot held around ex_div_date; compute credit = compute_franking_credit(aud_dividend_amount, effective_franking_pct)
       - Note: This is the most complex wiring step. If dividend currency != "AUD", apply FX conversion first.
    7. Build `TaxYearResult` list: `build_tax_year_results(all_disposed_lots, marginal_tax_rate)` — this gives CGT events and payable per year
    8. Merge franking credit data into TaxYearResults (update the per-year franking_credits_claimed and dividend_income fields)
    9. Compute after_tax_cagr: create an adjusted equity curve by subtracting tax payments on July 1 following each tax year end. Use the same cagr() function from metrics.py on the adjusted curve.
    10. Assemble and return TaxAwareResult(backtest=bt_result, tax=TaxSummary(...))

    **Important design constraints:**
    - `run_backtest()` is called first and unchanged — Phase 2 tests never touched
    - Tax engine sees only `list[Trade]` and DB queries — it never touches the price DataFrame
    - AUD tickers: cost_basis_usd=None, proceeds_usd=None throughout
    - Log at INFO level at start and end: ticker count, trade count, tax years found
