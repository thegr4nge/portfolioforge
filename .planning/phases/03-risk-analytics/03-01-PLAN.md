Create `src/portfolioforge/engines/risk.py` with pure computation functions (no I/O, no display). Follow the pattern in `engines/backtest.py`:

1. `compute_var_cvar(daily_returns: pd.Series, confidence: float = 0.95) -> dict[str, float]`:
   - Historical method: `var = np.percentile(daily_returns.dropna(), (1 - confidence) * 100)`
   - CVaR: mean of returns <= VaR. If no tail losses, CVaR = VaR.
   - Returns `{"var": var, "cvar": cvar}` as negative floats.

2. `compute_drawdown_periods(cumulative: pd.Series, top_n: int = 5) -> list[dict]`:
   - Use `cumulative.cummax()` to find running peaks.
   - Track contiguous drawdown regions (where cumulative / running_max - 1 < 0).
   - For each region: record peak_date, trough_date (idxmin of segment), recovery_date (where drawdown returns to 0), depth, duration_days, recovery_days.
   - Handle unrecovered drawdowns at end of series: `recovery_date=None`, `recovery_days=None`.
   - Sort by depth (most negative first), return top_n.
   - See RESEARCH.md code example for implementation reference.

3. `compute_correlation_matrix(prices: pd.DataFrame) -> pd.DataFrame`:
   - Compute daily returns via `prices.pct_change().dropna()`.
   - Return `daily_returns.corr()`.
   - If prices has < 2 columns, return empty DataFrame.

4. `compute_sector_exposure(tickers: list[str], weights: list[float], sectors: dict[str, str], concentration_threshold: float = 0.40) -> dict`:
   - Group weights by sector from the sectors mapping.
   - Generate warning strings for any sector exceeding threshold (skip "Unknown" sectors).
   - Return `{"breakdown": sector_weights, "warnings": warnings}`.
