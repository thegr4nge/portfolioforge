Create `src/portfolioforge/services/risk.py` following the pattern in `services/backtest.py`:

1. `run_risk_analysis(backtest_config: BacktestConfig) -> tuple[BacktestResult, RiskAnalysisResult]`:
   - Call `run_backtest(backtest_config)` to get BacktestResult (reuses existing service).
   - Build a `pd.Series` of portfolio cumulative returns from `result.portfolio_cumulative` with `result.dates` as index.
   - Compute daily returns from cumulative: `cumulative.pct_change().dropna()`.
   - Call `compute_var_cvar(daily_returns)` to get VaR/CVaR dict.
   - Call `compute_drawdown_periods(cumulative, top_n=5)` to get drawdown periods list.
   - Reconstruct the aligned price DataFrame from backtest data: re-fetch and align prices for individual tickers (use the same fetch pattern as services/backtest.py -- call `_fetch_all` and `align_price_data`). This is needed because BacktestResult only stores the combined portfolio series, not per-asset prices.
   - Call `compute_correlation_matrix(aligned_prices)` -- skip if only 1 ticker.
   - Build `RiskMetrics` from VaR/CVaR dict.
   - Build `DrawdownPeriod` models from the drawdown periods list.
   - Build `RiskAnalysisResult` with `risk_metrics`, `drawdown_periods`, correlation as nested dict (use `corr_df.to_dict()` to convert DataFrame to `dict[str, dict[str, float]]`), and `sector_exposure=None` (sector wired in Plan 03).
   - Return both BacktestResult and RiskAnalysisResult.

   Note: To avoid duplicating fetch logic, factor out the price fetching into a helper or import from services/backtest. The simplest approach: import `_fetch_all` from `services/backtest` (it's a module-level function, not a public API concern in this project). Or duplicate the 5-line fetch pattern inline.

Create `src/portfolioforge/output/risk.py` following the pattern in `output/backtest.py`:

1. `render_risk_analysis(backtest_result: BacktestResult, risk_result: RiskAnalysisResult, console: Console) -> None`:
   - First call `render_backtest_results(backtest_result, console)` to show the existing backtest output (performance summary + allocation table).
   - Then render risk-specific sections:

2. **Risk Metrics Panel**: A Rich Table titled "Risk Metrics" with two columns (Metric, Value):
   - "VaR (95% daily)" -> format as percentage with `_color_pct` style (red since negative): e.g. "-2.31%"
   - "CVaR (95% daily)" -> same format
   - Import `_color_pct` from `output/backtest.py` or duplicate the 5-line helper.

3. **Top Drawdown Periods Table**: Rich Table titled "Worst Drawdown Periods" with columns: Rank, Peak Date, Trough Date, Recovery Date, Depth, Duration (days), Recovery (days).
   - For each DrawdownPeriod in `risk_result.drawdown_periods`:
     - Rank: "1", "2", etc.
     - Peak/Trough dates formatted as YYYY-MM-DD.
     - Recovery Date: date string or "[yellow]Not recovered[/yellow]" if None.
     - Depth: format as percentage with red color (e.g. "[red]-25.3%[/red]").
     - Duration: integer days.
     - Recovery: integer days or "[yellow]N/A[/yellow]" if None.

4. **Correlation Matrix Table**: Rich Table titled "Asset Correlation Matrix".
   - Skip entirely if `risk_result.correlation_matrix` is empty (single-asset portfolio). Print "[dim]Correlation requires 2+ assets[/dim]" instead.
   - First column "" (row labels, bold), then one column per ticker.
   - For each row, color-code each cell:
     - value >= 0.8: `[red]` (high correlation = concentration risk)
     - value >= 0.5: `[yellow]`
     - value >= -0.5: `[green]` (good diversification)
     - value < -0.5: `[cyan]` (negative correlation = hedge)
   - Format values as "+0.85", "-0.23" etc (sign always shown, 2 decimal places).
