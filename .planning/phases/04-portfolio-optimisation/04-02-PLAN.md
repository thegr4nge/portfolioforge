  <name>Task 1: Create optimisation service layer</name>
  <files>src/portfolioforge/services/optimise.py</files>
  <action>
  Create `src/portfolioforge/services/optimise.py` following the pattern in `services/risk.py` and `services/backtest.py`.

  The service orchestrates: fetch prices -> align -> engine computation -> build result models.

  Import from existing modules:
  - `_fetch_all` from `services/backtest` (reuse, don't duplicate)
  - `align_price_data` from `engines/backtest`
  - `compute_optimal_weights`, `compute_efficient_frontier`, `score_portfolio` from `engines/optimise`
  - `PriceCache` from `data/cache`
  - All models from `models/optimise`

  Implement two functions:

  1. **`run_validate(config: OptimiseConfig) -> OptimiseResult`**
     - Fetch prices: `cache = PriceCache()`, `fx_cache = {}`, call `_fetch_all(config.tickers, config.period_years, cache, fx_cache)`
     - Align prices: `aligned = align_price_data([r.price_data for r in results if r.price_data])`
     - Set `weight_bounds = (config.min_weight, config.max_weight)`
     - Call `score_portfolio(aligned, config.weights, weight_bounds, config.risk_free_rate)` to get score dict
     - Call `compute_efficient_frontier(aligned, weight_bounds, risk_free_rate=config.risk_free_rate)` to get frontier points
     - Call `compute_optimal_weights(aligned, weight_bounds, config.risk_free_rate)` to get max-Sharpe portfolio
     - Build and return `OptimiseResult` with:
       - `mode="validate"`
       - `tickers=config.tickers`
       - `suggested_weights=score_dict["optimal_weights"]` (optimal at same risk)
       - `expected_return/volatility/sharpe_ratio` from the max-Sharpe optimal
       - `frontier_points` as list of `FrontierPoint` models
       - `score` as `PortfolioScore` model built from score_dict
       - `user_weights` as dict mapping tickers to weights

  2. **`run_suggest(config: OptimiseConfig) -> OptimiseResult`**
     - Same fetch + align flow
     - Call `compute_optimal_weights(aligned, weight_bounds, config.risk_free_rate)` to get optimal portfolio
     - Call `compute_efficient_frontier(aligned, weight_bounds, risk_free_rate=config.risk_free_rate)` for chart data
     - Build and return `OptimiseResult` with:
       - `mode="suggest"`
       - `tickers=config.tickers`
       - `suggested_weights=optimal["weights"]`
       - `expected_return/volatility/sharpe_ratio` from optimal
       - `frontier_points` as list of `FrontierPoint` models
       - `score=None`, `user_weights=None`

  NOTE: The research says optimise only needs prices for mu/S computation, NOT a full backtest. So fetch and align prices directly (like risk service does for correlation), don't call run_backtest.
  </action>

