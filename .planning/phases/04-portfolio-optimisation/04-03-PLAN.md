  Add `render_efficient_frontier_chart` function to `src/portfolioforge/output/optimise.py`.

  Follow the charting pattern from `output/backtest.py` (which uses `render_cumulative_chart` with plotext). Key patterns to match:
  - `import plotext as plt`
  - `plt.clear_figure()` before drawing
  - `plt.show()` at the end

  Implement `render_efficient_frontier_chart(result: OptimiseResult) -> None`:

  1. **Clear and set up:**
     ```python
     plt.clear_figure()
     plt.theme("dark")  # Match existing chart style if used in backtest chart
     ```

  2. **Plot the efficient frontier curve:**
     - Extract `vols = [p.volatility * 100 for p in result.frontier_points]`
     - Extract `rets = [p.expected_return * 100 for p in result.frontier_points]`
     - `plt.plot(vols, rets, label="Efficient Frontier", color="blue")`

  3. **Plot the max-Sharpe optimal portfolio:**
     - `plt.scatter([result.volatility * 100], [result.expected_return * 100], label="Optimal (Max Sharpe)", color="green", marker="diamond")`

  4. **Plot user's portfolio (validate mode only):**
     - If `result.score is not None`:
       - `plt.scatter([result.score.user_volatility * 100], [result.score.user_return * 100], label="Your Portfolio", color="red", marker="x")`

  5. **Set axis limits with padding** (Pitfall 5 from research: single-point scatter can be invisible):
     - Collect ALL x-values and y-values (frontier + optimal + user if exists)
     - Set `plt.xlim(min_x - padding, max_x + padding)` where padding = 0.5 (percentage points)
     - Set `plt.ylim(min_y - padding, max_y + padding)`

  6. **Labels and display:**
     ```python
     plt.title("Efficient Frontier")
     plt.xlabel("Volatility (%)")
     plt.ylabel("Expected Return (%)")
     plt.show()
     ```

  NOTE: plotext `scatter` with single-element lists works fine when axis limits are set explicitly. The marker types available in plotext are: "dot", "x", "hd", "fhd", "braille", "diamond", etc. Use "diamond" for optimal and "x" for user portfolio to distinguish them visually.
