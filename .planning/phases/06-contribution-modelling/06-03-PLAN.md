- Update ProjectionResult: add `total_contributed: float = 0.0` and `contribution_summary: str = ""` fields

**services/montecarlo.py** -- Update `run_projection`:
- Import `build_contribution_array` from engines.contribution and `ContributionSchedule` from models.contribution
- After step 4 (sigma scaling), before simulate_gbm call:
  - If `config.contribution_schedule` is not None and has_contributions:
    - Call `build_contribution_array(config.years, schedule.regular_amount, schedule.frequency, schedule.lump_sums)` to get contributions array
    - Pass `contributions=contrib_array` to `simulate_gbm` (instead of monthly_contribution)
    - Compute total_contributed = sum of contributions array
    - Build contribution_summary string: "Monthly: ${monthly_equiv:,.0f} ({frequency})" + any lump sums listed
  - Else: use existing `monthly_contribution` parameter path (backward compat)
- Pass total_contributed and contribution_summary to ProjectionResult

**output/montecarlo.py** -- Update `render_projection_results`:
- If `result.contribution_summary` is not empty, replace the simple "Monthly Contribution" row with:
  - "Contribution Plan" row showing the summary
  - "Total Contributed" row showing total_contributed formatted as currency
- This replaces the single monthly_contribution display when a full schedule is provided

**cli.py** -- Update `project` command to add contribution schedule options:
- Add `--frequency` option: `str` with default "monthly", choices: weekly, fortnightly, monthly. Only relevant when --contribution > 0.
- Add `--lump-sum` option: `list[str]` (repeatable), format "MONTH:AMOUNT" (e.g., "12:5000" means $5000 at month 12). Parse each into LumpSum(month=int, amount=float).
- When contribution > 0 OR lump_sums provided, build a `ContributionSchedule` and set it on `ProjectionConfig.contribution_schedule`
- Keep the existing `--contribution` option as the regular amount
- The existing `monthly_contribution` field on ProjectionConfig should be set to 0.0 when using contribution_schedule (to avoid double-counting)
- Parse lump-sum format with error handling: invalid format -> red error message -> Exit(1)
