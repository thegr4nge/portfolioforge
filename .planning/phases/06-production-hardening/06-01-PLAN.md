<objective>
Harden the tax engine core against three high-severity production risks identified in external review:
1. SMSF pension phase silently applies 0% tax (ECPI not implemented) — hard-block with NotImplementedError.
2. Tax engine output is not version-stamped — stamp TAX_ENGINE_VERSION into every TaxYearResult and the Word export Methodology section.
3. FX rate lookup crashes on weekends/public holidays — add 5-day prior-business-day fallback.

Purpose: These three fixes are required before any B2B client runs a real backtest. Pension phase silent miscalculation is a liability risk; missing version stamp is an audit risk; FX ValueError on a weekend crashes client sessions.

Output: Updated engine.py (pension guard + version import), updated models.py (version field on TaxYearResult), updated fx.py (fallback loop), updated exporter.py (version in Methodology table), updated streamlit_app.py (pension phase gate), updated tests/test_tax_engine.py (new tests for all three).
</objective>

    Step 1 — Rewrite get_aud_usd_rate() in fx.py:
    ```python
    from datetime import timedelta
    from loguru import logger

    _FX_FALLBACK_MAX_DAYS: int = 5  # covers 4-day Easter weekend plus one buffer

    def get_aud_usd_rate(conn: sqlite3.Connection, trade_date: date) -> float:
        for delta in range(_FX_FALLBACK_MAX_DAYS + 1):
            lookup_date = trade_date - timedelta(days=delta)
            row = conn.execute(_AUD_USD_SQL, (lookup_date.isoformat(),)).fetchone()
            if row is not None:
                if delta > 0:
                    logger.debug(
                        "FX rate for {} not found; using {} (T-{})",
                        trade_date, lookup_date, delta
                    )
                return float(row[0])
        raise ValueError(
            f"No AUD/USD FX rate found for {trade_date} or any of the "
            f"{_FX_FALLBACK_MAX_DAYS} prior calendar days. "
            "Re-ingest FX data for this period."
        )
    ```
