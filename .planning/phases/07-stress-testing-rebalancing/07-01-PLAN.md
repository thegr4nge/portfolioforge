Create `models/stress.py` with Pydantic models:

- `StressScenario(BaseModel)`: name (str), start_date (date), end_date (date), scenario_type (Literal["historical", "custom"]), shock_sector (str | None = None), shock_pct (float | None = None)
- `StressConfig(BaseModel)`: tickers (list[str]), weights (list[float]), scenarios (list[StressScenario]), period_years (int = 20). Add model_validator to check tickers/weights length match and weights sum to ~1.0 (same pattern as BacktestConfig).
- `ScenarioResult(BaseModel)`: scenario_name (str), start_date (date), end_date (date), portfolio_drawdown (float), recovery_days (int | None), portfolio_return (float), per_asset_impact (dict[str, float])
- `StressResult(BaseModel)`: portfolio_name (str), scenarios (list[ScenarioResult])

Create `engines/stress.py` with:

1. `HISTORICAL_SCENARIOS` dict constant mapping scenario names to (start_date, end_date) tuples:
   - "2008 GFC": (date(2007, 10, 9), date(2009, 3, 9))
   - "2020 COVID": (date(2020, 2, 19), date(2020, 3, 23))
   - "2022 Rate Hikes": (date(2022, 1, 3), date(2022, 10, 12))

2. `apply_historical_scenario(prices: pd.DataFrame, weights: np.ndarray, start_date: date, end_date: date) -> dict`:
   - Filter prices to date range using `prices.index.date >= start_date` and `<= end_date`
   - If fewer than 5 trading days in window, raise ValueError with clear message about insufficient data coverage
   - Call `compute_cumulative_returns(crisis_prices, weights, None)` from engines/backtest
   - Call `compute_drawdown_periods(cumulative, top_n=1)` from engines/risk to get worst drawdown
   - Compute per-asset impact: `(crisis_prices.iloc[-1] / crisis_prices.iloc[0] - 1)` for each ticker
   - Compute portfolio_return from cumulative series: `cumulative.iloc[-1] / cumulative.iloc[0] - 1`
   - Return dict with: portfolio_drawdown, recovery_days (from drawdown period, None if unrecovered), portfolio_return, per_asset_impact

3. `apply_custom_shock(prices: pd.DataFrame, weights: np.ndarray, sectors: dict[str, str], shock_sector: str, shock_pct: float) -> dict`:
   - Apply instantaneous shock (NOT distributed): multiply affected ticker prices by (1 + shock_pct) from midpoint of series onwards
   - Identify shocked tickers: `[t for t in prices.columns if sectors.get(t, "").lower() == shock_sector.lower()]`
   - If no tickers match the sector, raise ValueError with message listing available sectors
   - Create modified prices DataFrame: copy prices, for shocked tickers multiply all rows from midpoint onwards by (1 + shock_pct)
   - Compute cumulative returns on modified prices using `compute_cumulative_returns`
   - Compute drawdown using `compute_drawdown_periods`
   - Return same dict structure as historical scenario

Follow existing engine pattern: no I/O, no display imports, pure computation with pandas/numpy inputs.
