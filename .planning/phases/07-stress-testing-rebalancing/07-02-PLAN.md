Create `models/rebalance.py` with Pydantic models:

- `DriftSnapshot(BaseModel)`: date (date), actual_weights (dict[str, float]), target_weights (dict[str, float]), max_drift (float)
- `TradeItem(BaseModel)`: ticker (str), action (Literal["BUY", "SELL"]), weight_change (float), dollar_amount (float | None = None)
- `StrategyComparison(BaseModel)`: strategy_name (str), total_return (float), annualised_return (float), max_drawdown (float), volatility (float), sharpe_ratio (float), rebalance_count (int)
- `RebalanceConfig(BaseModel)`: tickers (list[str]), weights (list[float]), period_years (int = 10, threshold (float = 0.05), portfolio_value (float | None = None). Add model_validator for tickers/weights match and weights sum ~1.0.
- `RebalanceResult(BaseModel)`: portfolio_name (str), drift_snapshots (list[DriftSnapshot]), trades (list[TradeItem]), strategy_comparisons (list[StrategyComparison]), current_weights (list[float])

Create `engines/rebalance.py` with pure computation functions:

1. `compute_weight_drift(prices: pd.DataFrame, target_weights: np.ndarray, check_freq: str = "MS") -> list[dict]`:
   - Step through daily returns, track current_weights using the same drift logic as `compute_cumulative_returns` (multiply by (1+return), renormalize)
   - At each checkpoint date (resample by check_freq), record: date, actual weights (dict ticker->float), target weights (dict), max_drift (max absolute deviation)
   - Return list of snapshot dicts

2. `generate_trade_list(tickers: list[str], current_weights: np.ndarray, target_weights: np.ndarray, portfolio_value: float | None = None) -> list[dict]`:
   - For each ticker, compute delta = target - current
   - Skip trivial drift (abs(delta) < 0.001)
   - Create trade dict: ticker, action ("BUY" if delta > 0 else "SELL"), weight_change (abs), dollar_amount (abs(delta) * portfolio_value if provided, else None)
   - Sort by abs(weight_change) descending
   - Return list of trade dicts

3. `compute_cumulative_with_threshold(prices: pd.DataFrame, weights: np.ndarray, threshold: float) -> tuple[pd.Series, int]`:
   - Same structure as the rebalancing loop in compute_cumulative_returns, but instead of calendar dates, check `max(abs(current_weights - weights)) > threshold` at each step
   - If drift exceeds threshold, reset current_weights to target and increment rebalance_count
   - Return (cumulative_series, rebalance_count)

4. `compare_rebalancing_strategies(prices: pd.DataFrame, weights: np.ndarray, threshold: float = 0.05) -> list[dict]`:
   - Run compute_cumulative_returns with each frequency: {"Never": None, "Monthly": "MS", "Quarterly": "QS", "Annually": "YS"} -- import from engines/backtest
   - Also run compute_cumulative_with_threshold for threshold-based strategy
   - For each result, call compute_metrics (from engines/backtest) to get return/drawdown/sharpe/etc
   - Count rebalance events: for calendar strategies, count resample dates; for threshold, use returned count; for never, 0
   - Return list of dicts with: strategy_name, total_return, annualised_return, max_drawdown, volatility, sharpe_ratio, rebalance_count
