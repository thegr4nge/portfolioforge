  <action>
Create `services/rebalance.py` following the established service pattern:

- `run_rebalance_analysis(config: RebalanceConfig) -> RebalanceResult`:
  - Create PriceCache and fx_cache dict
  - Call `_fetch_all(config.tickers, config.period_years, cache, fx_cache)` imported from services/backtest
  - Align prices using `align_price_data` from engines/backtest
  - weights = np.array(config.weights)
  - Call `compute_weight_drift(aligned, weights)` -> list of snapshot dicts
  - Call `compute_final_weights(aligned, weights)` from engines/backtest to get current_weights
  - Call `generate_trade_list(config.tickers, np.array(current_weights), weights, config.portfolio_value)` -> list of trade dicts
  - Call `compare_rebalancing_strategies(aligned, weights, config.threshold)` -> list of strategy dicts
  - Convert raw dicts to Pydantic models: DriftSnapshot, TradeItem, StrategyComparison
  - Build portfolio_name from ticker:weight pairs (same pattern as backtest service)
  - Return RebalanceResult

Create `output/rebalance.py` with `render_rebalance_results(result: RebalanceResult) -> None`:

1. Print Rich Panel header: "Rebalancing Analysis" for portfolio_name

2. **Drift Summary Table**: Show first and last 3 drift snapshots (to avoid huge tables for long periods). Columns: Date, Max Drift, then one column per ticker showing actual weight. Use color: green if drift < 2%, yellow if 2-5%, red if > 5%.

3. **Trade Recommendations Table**: Title "Recommended Trades to Rebalance". Columns: Ticker, Action (green "BUY" / red "SELL"), Weight Change (formatted as +/-%), Dollar Amount (if available, formatted with $ and commas). If no trades needed, print "[green]Portfolio is within tolerance[/green]".

4. **Strategy Comparison Table**: Title "Rebalancing Strategy Comparison". Columns: Strategy, Total Return, Annualised Return, Max Drawdown, Volatility, Sharpe, Trades. Use _color_pct helper (import from output/backtest or duplicate) for return/drawdown values. Highlight the best Sharpe row with bold.

Use Rich Console, Table, Panel -- same patterns as output/risk.py and output/backtest.py.
