  <action>
Create `src/portfolioforge/engines/explain.py` as a pure Python module with:

1. A `_METRIC_EXPLANATIONS` dictionary mapping metric keys to explanation templates with threshold-based qualifiers. Use the exact template and threshold data from the RESEARCH.md "Explanation Engine Pattern" and "Code Examples" sections. Cover ALL metrics from the "Complete Metric Inventory" in RESEARCH.md:

   - `sharpe_ratio`, `sortino_ratio`, `max_drawdown`, `volatility`, `annualised_return`, `total_return` (from BacktestResult/StrategyComparison)
   - `var_95`, `cvar_95` (from RiskMetrics)
   - `efficiency_ratio` (from PortfolioScore)
   - `correlation` (for pairwise correlation values)
   - `probability` (from GoalAnalysis)
   - `drawdown_depth` (for DrawdownPeriod.depth -- reuse max_drawdown thresholds)
   - `stress_drawdown` (for ScenarioResult.portfolio_drawdown -- "During {scenario}, your portfolio would have dropped {value:.1%}. {qualifier}")
   - `lump_win_pct` (from CompareResult -- "Lump sum investing beat DCA in {value:.0%} of historical periods. {qualifier}" with thresholds at 0.7/0.5)
   - `rebalance_count` (for StrategyComparison -- "{value:.0f} rebalancing events over the period." with no thresholds, informational only)

2. `explain_metric(key: str, value: float) -> str | None` -- returns formatted explanation or None if key unknown

3. `explain_all(metrics: dict[str, float]) -> dict[str, str]` -- batch explanation for a dict of metrics, skipping unknowns

Thresholds are checked top-to-bottom: value >= threshold selects that qualifier. Use `float("-inf")` as the catch-all final threshold.

For `total_return` and `rebalance_count`, use empty thresholds list (informational, no qualifier).

Create `tests/portfolioforge/test_explain_engine.py` with tests:
- `test_explain_metric_known_key`: sharpe_ratio with value 1.2 returns string containing "1.20" and "Good"
- `test_explain_metric_unknown_key`: returns None for "unknown_metric"
- `test_explain_metric_threshold_boundaries`: test that sharpe=0.4 gets "Below average", sharpe=0.5 gets "Average", sharpe=1.0 gets "Good", sharpe=1.5 gets "Excellent"
- `test_explain_metric_negative_values`: max_drawdown with -0.15 returns string containing "Moderate"
- `test_explain_all_filters_unknowns`: pass dict with known and unknown keys, verify only known keys in result
- `test_explain_metric_no_thresholds`: total_return returns string without qualifier text (just the template with empty qualifier)

Run: `cd ~/uni-projects && source .venv/bin/activate && python -m pytest tests/portfolioforge/test_explain_engine.py -v`

**Pattern for all output modules:** Add an `explain: bool = True` parameter to each main render function. When `explain=True`, after rendering the existing metrics table(s), add a Rich Panel containing explanation text for the key metrics displayed. Use `from portfolioforge.engines.explain import explain_metric`. Render explanations using `console.print(Panel(explanation_text, title="What This Means", border_style="dim"))` with `highlight=False` to prevent Rich markup interpretation of brackets in explanation text.

IMPORTANT: Do NOT inline explanations in table cells. Show them in a separate panel BELOW each metrics table. This follows the research recommendation to avoid verbosity overwhelming the output.

**output/backtest.py** -- modify `render_backtest_results(result, console, *, explain=True)`:
- After the Performance Summary table, if `explain=True`, build explanation text for: `total_return`, `annualised_return`, `max_drawdown`, `volatility`, `sharpe_ratio`, `sortino_ratio` using `explain_metric()` with values from `result`.
- Join explanations with newlines, render in a Panel with title="What This Means" and border_style="dim".
- Use `console.print(Panel(Text(explanation_text), ...))` wrapping in `rich.text.Text` to avoid markup interpretation. Import `Text` from `rich.text`.

**output/risk.py** -- modify `render_risk_analysis(backtest_result, risk_result, console, *, explain=True)`:
- Pass `explain=explain` through to `render_backtest_results()` call.
- After the Risk Metrics table, if `explain=True`, add explanations for `var_95` and `cvar_95`.
- After the correlation matrix, if `explain=True` and there are 2+ assets, explain the highest off-diagonal correlation using the `correlation` key.

**output/optimise.py** -- modify both `render_suggest_results(result, console, *, explain=True)` and `render_validate_results(result, console, *, explain=True)`:
- After metrics display, explain `expected_return` (use `annualised_return` key), `volatility`, `sharpe_ratio`.
- For validate mode, also explain `efficiency_ratio` from `result.score`.

**output/montecarlo.py** -- modify `render_projection_results(result, console, *, explain=True)`:
- Explain the `probability` metric if `result.goal` exists.
- Explain `volatility` (using `result.sigma`) and `annualised_return` (using `result.mu`).

**output/contribution.py** -- modify `render_compare_results(result, console, *, explain=True)`:
- Explain `lump_win_pct` from the result.

**output/stress.py** -- modify `render_stress_results(result, console, *, explain=True)`:
- After each scenario row or after the full table, explain the worst `stress_drawdown` value across scenarios.

**output/rebalance.py** -- modify `render_rebalance_results(result, console, *, explain=True)`:
- After strategy comparison table, explain `sharpe_ratio` and `max_drawdown` for the best-performing strategy.

**CLI changes (cli.py):**
- Add `explain: Annotated[bool, typer.Option("--explain/--no-explain", help="Show plain-English metric explanations")] = True` parameter to these commands: `backtest`, `analyse`, `suggest`, `validate`, `project`, `compare`, `stress_test`, `rebalance`.
- Pass `explain=explain` to each render function call.
- For `render_cumulative_chart` and other chart-only functions, no change needed (charts don't show explanations).

Keep all existing behavior intact. The `explain` parameter defaults to `True` so explanations show by default (per UX-04 requirement). Users opt out with `--no-explain`.
