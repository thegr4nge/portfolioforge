Questions

Common questions

Answers for accountants evaluating PortfolioForge for their SMSF practice.

Yes. The engine applies the SMSF-specific 33.33% CGT discount (not the 50% rate available to individuals), enforces the 45-day holding rule for franking credits, and uses the Australian financial year (1 July – 30 June) for all event bucketing. The calculation methodology is validated against ATO published worked examples and documented in every report's Methodology section. The tax engine version is stamped into each output for audit traceability.

FIFO (First In, First Out), which is the default and most common method accepted by the ATO. When a parcel is sold, the engine matches it against the earliest-purchased lots first and calculates the cost basis for those specific parcels. The full lot-matching log is included in every report so the accountant and auditor can verify each disposal. An optional highest-cost method is also available via the CLI for cases where the trustee has made a written election.

Accumulation phase is fully supported. Pension phase — which involves the Exempt Current Pension Income (ECPI) calculation requiring an actuarial certificate — is currently not implemented. Attempting to run a pension phase analysis raises a clear error rather than silently producing an incorrect result. ECPI support is on the roadmap and will require the practice to supply the actuarial ECPI percentage for the relevant years.
